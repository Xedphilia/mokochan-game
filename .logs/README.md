# .logs/ - 判断ログ

Codex（ローカルLLM）が作業した際の判断・変更内容を記録する。
npaka123「Codexゲーム開発プロンプトまとめ」のワークフローに基づく。

## ファイル命名規則
`<機能名>_YYYY-MM-DD.md`

例:
- `food_tab_20260503.md`
- `gacha_animation_20260504.md`

## テンプレート
```markdown
# [機能名] 作業ログ - YYYY-MM-DD

## 実装内容
- 変更した関数/セクション:
- 新規追加した関数:
- 変更の理由:

## 検討した別案
- 案A:（採用しなかった理由）
- 案B:（採用した）

## 注意事項
- ここを触るときは○○に気をつける

## テスト確認
- [ ] Playwright でタブを開いて動作確認
- [ ] localStorage の整合性確認
- [ ] スマホサイズで表示確認
```
